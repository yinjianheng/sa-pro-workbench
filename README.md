## 🎯 为什么选择这个 Skill？

| 痛点 | 解决方案 |
|------|---------|
| 📐 架构图绘制耗时，风格不统一 | **13 类专业图表**：C4 模型 · DFD · BPMN · 部署架构……输出 .drawio 可编辑源文件 |
| 📊 PPT 汇报制作低效，审美疲劳 | **PPT 工场**：5 类汇报 PPT + 6 套配色方案，一键生成 |
| 📝 方案文档结构混乱，缺乏标准 | **文档工场**：HLD 12 章 · LLD 10 章 · SOW 8 章标准结构 |
| 🎯 投标 RFP 响应效率低，质量参差 | **RFP 逐条响应**：智能匹配方案能力，自动生成应答内容 |
| 🗣️ 客户沟通缺乏预判和策略 | **SPIN 需求挖掘 + 客户沟通预判与应答策略手册** |

## 🧠 方法论基础

| 方法论 | 说明 |
|--------|------|
| C4 模型 | 系统上下文 → 容器 → 组件，支持"地图式缩放" |
| 4+1 视图 | 逻辑/开发/进程/物理/场景 |
| TOGAF 4A + ADM 9 阶段 | 业务→应用→数据→技术，完整企业架构生命周期 |
| ArchiMate 3.2 | The Open Group 企业架构建模标准 |
| Wardley Mapping | 价值链四阶段演变战略决策框架 |
| FinOps | 云财务管理三阶段循环 |
| 零信任安全架构 | "永不信任，始终验证" |
| SPIN 需求挖掘 | 情境→问题→影响→需求确认 |
| ADR 架构决策记录 | 结构化架构决策文档 |

## 📦 快速开始

```bash
cp -r sa-pro-workbench ~/.claude/skills/
```

**触发方式**：直接描述需求，支持 40+ 中文别名和 10+ 英文别名：
- "帮我画一个系统架构图"
- "写一份框架方案 HLD"
- "做投标述标 PPT"
- "逐条响应这份 RFP"
- "做竞品分析报告"
